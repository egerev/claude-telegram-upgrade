# Cloud Server Setup for Claude Code + Telegram

Complete guide to running Claude Code with Telegram integration on a cloud server.

## Architecture

```
Your Computer (Tailscale exit node)
    ↑ all traffic exits through your home IP
    │
Cloud Server (Tailscale client)
    ├── tmux: claude-code (project-1) ← Telegram bot 1
    ├── tmux: claude-code (project-2) ← Telegram bot 2
    ├── codex (optional)
    └── tailscale --exit-node=your-computer
```

## Key Discovery: Authentication

Claude Code subscription auth on headless servers requires the `CLAUDE_CODE_OAUTH_TOKEN` environment variable. This is a long-lived OAuth token generated by `claude setup-token` on a machine with browser access.

### How to get the token

On your local machine (Mac/PC with browser):

```bash
claude setup-token
```

This opens a browser, you authenticate, and it saves a token to `~/.claude/.setup-token`. This token is valid for ~1 year.

### How to use on server

Store the token in a dedicated env file:
```bash
echo "export CLAUDE_CODE_OAUTH_TOKEN=$(cat ~/.claude/.setup-token | tr -d '[:space:]')" > ~/.claude/.env
chmod 600 ~/.claude/.env
```

Then source it before launching Claude Code:
```bash
source ~/.claude/.env
claude --channels plugin:telegram@claude-plugins-official --dangerously-skip-permissions
```

**Important:** Do NOT set `ANTHROPIC_API_KEY` — it overrides OAuth and switches to API billing. The channels feature (Telegram) only works with subscription auth.

## Launching Claude Code with Telegram

```bash
source ~/.claude/.env
claude \
  --channels plugin:telegram@claude-plugins-official \
  --dangerously-skip-permissions \
  --effort high \
  --model sonnet
```

Flags:
- `--channels plugin:telegram@...` — enables the Telegram MCP plugin as a channel
- `--dangerously-skip-permissions` — no permission prompts (headless)
- `--effort high` — higher reasoning effort
- `--model sonnet` (or `--model opus` for Max plan users)

## Multi-Bot Setup

Each project gets its own Telegram bot, state directory, and tmux session.

### Directory structure

```
~/.claude/channels/
├── telegram-project1/    ← bot for project 1
│   ├── .env              ← TELEGRAM_BOT_TOKEN=...
│   └── access.json       ← pairing/allowlist config
├── telegram-project2/    ← bot for project 2
│   ├── .env
│   └── access.json
```

### Per-project config

Each project has `.claude/settings.local.json`:

```json
{
  "env": {
    "TELEGRAM_STATE_DIR": "/home/ubuntu/.claude/channels/telegram-project1",
    "TELEGRAM_WHISPER_MODEL": "/home/ubuntu/.local/share/whisper-models/ggml-small.bin"
  }
}
```

### tmux sessions

```bash
# Create session for each project
tmux new-session -d -s project1 -c ~/projects/project1
tmux new-session -d -s project2 -c ~/projects/project2

# Launch Claude Code in each (token is loaded from ~/.claude/.env)
tmux send-keys -t project1 "source ~/.claude/.env && export PATH=\"\$HOME/.bun/bin:\$PATH\" && claude --channels plugin:telegram@claude-plugins-official --dangerously-skip-permissions --effort high --model sonnet" Enter

tmux send-keys -t project2 "source ~/.claude/.env && export PATH=\"\$HOME/.bun/bin:\$PATH\" && claude --channels plugin:telegram@claude-plugins-official --dangerously-skip-permissions --effort high --model sonnet" Enter
```

### Attach to sessions

```bash
TERM=xterm-256color tmux attach -t project1
# Ctrl-b d to detach
TERM=xterm-256color tmux attach -t project2
```

## Tailscale Exit Node (optional)

Routes all server traffic through your home IP so Claude Code sees your residential IP, not a datacenter.

### On your home machine

```bash
sudo tailscale up --advertise-exit-node
# Then approve in Tailscale admin console
```

### On the server

```bash
sudo tailscale up --authkey=tskey-auth-xxxxx
sudo tailscale set --exit-node=<home-machine-tailscale-ip>
```

**Note:** After setting exit node, SSH via public IP breaks. Use Tailscale IP instead:

```bash
ssh user@<server-tailscale-ip>
```

## Voice Transcription

The voice-transcription patch auto-transcribes Telegram voice messages. On a Linux server, use whisper-cpp:

```bash
# Install
sudo apt install cmake build-essential ffmpeg
git clone --depth 1 https://github.com/ggerganov/whisper.cpp.git /tmp/whisper.cpp
cd /tmp/whisper.cpp && cmake -B build && cmake --build build --config Release -j2
sudo cp build/bin/whisper-cli /usr/local/bin/whisper-cpp

# Download model
mkdir -p ~/.local/share/whisper-models
curl -L -o ~/.local/share/whisper-models/ggml-small.bin \
  "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin"
```

Models: `tiny` (75MB, fast, bad), `small` (466MB, balanced), `medium` (1.5GB, good Russian), `large-v3` (3GB, best).

## Plugin Locations

Claude Code stores the Telegram plugin in TWO locations:
- `~/.claude/plugins/cache/claude-plugins-official/telegram/*/`
- `~/.claude/plugins/marketplaces/claude-plugins-official/external_plugins/telegram/`

**Patches must be applied to BOTH.** Claude Code may run `server.ts` from either location.

## Troubleshooting

### "Channels require claude.ai authentication"
`ANTHROPIC_API_KEY` is set and overriding OAuth. Remove it and use `CLAUDE_CODE_OAUTH_TOKEN` instead.

### "409 Conflict" on Telegram
Another bot instance is using the same token. Kill old processes: `pkill -f "bun server.ts"`

### tmux: "missing or unsuitable terminal: xterm-ghostty"
Use: `TERM=xterm-256color tmux attach -t session-name`

### First launch asks for theme selection
Interactive — must attach to tmux and press Enter once. After that, headless works fine.
